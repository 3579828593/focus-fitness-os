// 修复后: scrypt 哈希 + timingSafeEqual
users = [{ username: 'admin', password_hash: 'ad19b5...:c6ff7b...' }];
function verifyPassword(password, storedHash) {
    var parts = storedHash.split(':');
    var salt = Buffer.from(parts[0], 'hex');
    var hash = Buffer.from(parts[1], 'hex');
    var computedHash = crypto.scryptSync(password, salt, 64);
    return crypto.timingSafeEqual(hash, computedHash);
}

关键经验教训

GitHub Actions YAML 编码陷阱：中文注释会导致 GitHub Actions 工作流解析失败，错误信息为 "workflow file issue"，但不显示具体原因。解决方案：.github/workflows/ 目录下的 YAML 文件仅使用英文注释。

Node-RED function 沙箱安全改进：使用 Node.js 内置 crypto.scryptSync + crypto.timingSafeEqual 替代外部 bcryptjs 依赖，减少镜像体积且无需修改 package.json。

文件持久化作为 SQLite 过渡方案：Refresh Token 使用 fs.writeFileSync 持久化到 JSON 文件，在 SQLite 完全配置前作为可靠的过渡方案。
